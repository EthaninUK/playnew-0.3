# 玩法交易所设计方案 v1.0

## 一、核心概念重新定义

### 1.1 产品定位
**Web3 玩法的"闲鱼+知识星球"** - 让用户之间、平台与用户之间自由交换玩法

**核心差异**:
- ❌ 不是传统商城(Marketplace):平台作为卖家,用户购买商品
- ✅ 是交易所(Exchange):用户之间交换玩法,平台提供撮合+担保

### 1.2 交易所核心逻辑

```
┌─────────────────────────────────────────────────────────┐
│                    玩法交易所生态                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  [供给方]                                    [需求方]    │
│                                                         │
│  1️⃣ 平台自营 (AI + 编辑团队)                             │
│     → 提供基础玩法库                                     │
│     → 免费 + 付费混合                                    │
│                                                         │
│  2️⃣ 用户贡献者 (UGC)                                     │
│     → 上传自己的玩法/工具/数据                            │
│     → 设置价格 or 交换条件                               │
│                                                         │
│  3️⃣ 专业玩家/KOL                                        │
│     → 售卖高级策略/信号/服务                              │
│     → 订阅制 or 按次付费                                 │
│                                                         │
│  ↓                                                      │
│  [交易方式]                                              │
│                                                         │
│  💰 直接购买: 用积分/现金购买玩法                         │
│  🔄 玩法互换: 用自己的玩法换别人的玩法                     │
│  🎁 免费分享: 贡献玩法获得积分/声望                       │
│  📦 打包售卖: 多个玩法组合成套餐                          │
│  🔔 订阅制: 持续获得最新玩法更新                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 二、核心交易模式设计

### 模式 1: 积分购买(最简单)
**适用场景**: 用户想快速获取某个玩法

```
用户 A 浏览玩法库
  ↓
发现感兴趣的玩法(需 50 积分)
  ↓
确认购买(扣除 50 积分)
  ↓
立即解锁查看完整内容
  ↓
玩法发布者获得 45 积分(平台抽成 10%)
```

**优势**: 简单直接,类似现有会员体系
**实现成本**: 低(复用现有积分系统)

---

### 模式 2: 玩法互换(创新点 🌟)
**适用场景**: 用户有自己的玩法,想换别人的玩法

```
用户 A 发布玩法 "LayerZero 空投攻略"(标价 100 积分)
  ↓
用户 B 想要这个玩法,但积分不足
  ↓
用户 B 发布玩法 "Arbitrum 生态挖矿指南"(标价 100 积分)
  ↓
平台匹配:两个玩法价值相近
  ↓
用户 A、B 互换玩法(双方都解锁对方内容)
  ↓
平台收取少量手续费(各扣 5 积分)
```

**优势**:
- 降低门槛:没钱也能换到玩法
- 激励 UGC:鼓励用户分享自己的经验
- 社区活跃:促进用户互动

**关键机制**:
1. **价值评估**: 玩法发布时需标明价值(积分)
2. **匹配算法**: 平台自动匹配价值相近的交换需求
3. **质量保障**: 互换后有评价系统,低质内容会被标记

---

### 模式 3: 悬赏求玩法(需求驱动)
**适用场景**: 用户有具体需求,找不到现成玩法

```
用户 A 发布悬赏 "求 zkSync Era 最新空投攻略"(悬赏 200 积分)
  ↓
用户 B 看到悬赏,撰写攻略并提交
  ↓
用户 A 查看并验收(满意/不满意)
  ↓
如满意:200 积分转给用户 B,平台抽成 10%
如不满意:用户 A 可拒绝,或由平台仲裁
```

**优势**:
- 需求驱动:让市场决定生产什么内容
- 快速响应:比平台自营更灵活
- 专家脱颖而出:高质量玩家获得更多悬赏

---

### 模式 4: 订阅制(持续更新)
**适用场景**: 专业玩家提供持续服务

```
KOL "空投猎人 Pro" 开设订阅服务
  ↓
订阅套餐:
  - 轻量版: 50 积分/月(周报 + 社群)
  - 专业版: 150 积分/月(日报 + 独家策略 + 1v1 答疑)
  - VIP 版: 500 积分/月(实时信号 + 代操作服务)
  ↓
用户选择订阅,每月自动扣积分
  ↓
KOL 持续提供内容,订阅者自动续费
```

**优势**:
- 稳定收入:KOL 可获得持续收入
- 高价值:用户获得持续更新的信息
- 社区沉淀:形成小圈子文化

---

### 模式 5: 打包售卖(套餐优惠)
**适用场景**: 同一主题的多个玩法

```
用户发布"空投大礼包"(包含 10 个玩法)
  ↓
单独购买总价: 500 积分
打包购买: 350 积分(7 折)
  ↓
用户一次性解锁 10 个玩法
```

**优势**:
- 提高客单价
- 提供价值感
- 清库存(老玩法可以打包重新售卖)

## 三、用户角色与激励体系

### 3.1 用户角色分层

| 角色 | 定义 | 权益 | 获取方式 |
|------|------|------|---------|
| **游客** | 未登录用户 | 只能浏览免费玩法列表 | - |
| **普通用户** | 注册用户 | • 初始 50 积分<br>• 可购买玩法<br>• 可发布玩法(需审核) | 注册即可 |
| **认证玩家** | 贡献高质量玩法 | • 玩法审核优先通过<br>• 分成比例提高(90%)<br>• 专属徽章 | 贡献 3 个 4.5 分以上玩法 |
| **专业 KOL** | 持续输出内容 | • 可开设订阅服务<br>• 可设置私密社群<br>• 平台推荐位 | 粉丝 ≥ 100 或月收入 ≥ 1000 积分 |
| **平台自营** | 官方账号 | • 发布基础玩法库<br>• 100% 收入归平台 | AI + 编辑团队 |

---

### 3.2 积分经济系统

#### 积分获取方式

| 行为 | 获得积分 | 频率限制 | 说明 |
|------|----------|---------|------|
| **注册账号** | +50 | 一次性 | 新用户福利 |
| **每日登录** | +5 | 每天 1 次 | 基础奖励 |
| **发布玩法** | +20-200 | 无限制 | 根据质量评分浮动 |
| **玩法被购买** | +90%(售价) | 无限制 | 平台抽成 10% |
| **玩法被互换** | 对方玩法价值 | 无限制 | 双向解锁 |
| **完成悬赏** | 悬赏金额 × 90% | 无限制 | 平台抽成 10% |
| **评价他人玩法** | +2 | 每天 10 次 | 有效评价(≥20 字) |
| **举报低质内容** | +20 | 审核通过 | 维护社区质量 |
| **邀请好友** | +30 | 每人 1 次 | 好友完成注册并活跃 3 天 |
| **购买会员** | 一次性大量积分 | 无限制 | $19 = 500 积分 |

#### 积分消耗方式

| 行为 | 消耗积分 | 说明 |
|------|----------|------|
| **购买玩法** | 5-500 | 由发布者设定 |
| **订阅 KOL** | 50-500/月 | 持续扣费 |
| **置顶玩法** | 100/7 天 | 增加曝光 |
| **VIP 社群** | 200-1000 | 加入私密社群 |
| **1v1 咨询** | 100-500/次 | 专家答疑 |

---

### 3.3 声望系统(防止刷单)

除了积分,增加"声望值"作为质量指标:

```
声望值计算公式:
声望 = (发布玩法数 × 10)
      + (平均评分 × 50)
      + (总销量 × 2)
      - (退款次数 × 20)
      - (被举报次数 × 50)
```

**声望等级**:
- 🌱 新手(0-100):发布玩法需审核
- 🌿 熟练(101-500):审核优先通过
- 🌳 专家(501-2000):可开设订阅服务
- 🏆 大师(2000+):平台推荐位,分成比例 95%

## 四、平台运营策略

### 4.1 冷启动阶段(0-3 个月)

**目标**: 验证"玩法互换"模式,建立初始内容池

#### 阶段 1: 平台自营打底(前 2 周)
- AI + 编辑团队生产 **50 个基础玩法**
- 涵盖 9 大主题(空投、DeFi、套利等)
- 30% 免费,70% 付费(5-50 积分)
- 目的:让用户有内容可换

#### 阶段 2: 种子用户招募(第 3-4 周)
- 邀请 **20-30 个真实玩家**(KOL、社群活跃用户)
- 给予初始 **500 积分 + 认证玩家身份**
- 要求:每人贡献 ≥3 个原创玩法
- 奖励:前 10 名贡献者额外 1000 积分

#### 阶段 3: 公测上线(第 5-8 周)
- 开放注册,每日限量 100 人(制造稀缺感)
- 推广策略:
  - Twitter/TG 社群宣传
  - "邀请 3 人得 100 积分"裂变
  - 与项目方合作发布悬赏任务
- 目标:
  - 注册用户 ≥500
  - UGC 玩法 ≥100
  - 日活用户 ≥50

#### 阶段 4: 数据复盘(第 9-12 周)
**关键指标**:
- 玩法互换 vs 积分购买比例(理想 1:3)
- 用户留存率(D7 留存 ≥30%)
- UGC 质量(平均评分 ≥4.0)
- 退款率(≤5%)

**决策点**:
- ✅ 如果互换模式受欢迎,继续扩大
- ❌ 如果用户更倾向购买,则弱化互换,专注积分商城

---

### 4.2 成长期(3-6 个月)

**目标**: 形成自循环的 UGC 生态

#### 策略 1: KOL 订阅服务
- 邀请 **10-20 个专业玩家**开设订阅服务
- 平台推广:首页推荐位、专题页
- 激励:首月订阅收入,平台 0 抽成(100% 归 KOL)

#### 策略 2: 悬赏市场
- 与项目方合作,发布官方悬赏任务
- 例如:"Optimism 生态玩法攻略"(悬赏 5000 积分)
- 吸引高质量创作者

#### 策略 3: 社区自治
- 建立"玩法评审委员会"(10-20 个高声望用户)
- 参与:低质内容下架投票、纠纷仲裁
- 奖励:每月固定积分补贴

---

### 4.3 成熟期(6-12 个月)

**目标**: 平台品牌化,吸引外部流量

#### 策略 1: 数据 API 开放
- 允许第三方接入玩法库(需付费)
- 例如:其他 DApp 嵌入"推荐玩法"模块
- 收费:按调用次数($0.01/次)

#### 策略 2: 白标服务
- 为项目方/DAO 提供定制版"玩法交易所"
- 收费:$5000-$20000 一次性 + $500/月维护

#### 策略 3: 线下活动
- 举办"玩法大赛",吸引 KOL 参赛
- 奖金池:$10,000-$50,000
- 曝光:Twitter Space、YouTube 直播

## 五、技术实现方案

### 5.1 数据库设计(基于现有系统扩展)

#### 扩展 `strategies` 表(玩法表)
```sql
ALTER TABLE strategies ADD COLUMN seller_id UUID REFERENCES auth.users(id);
-- 发布者 ID(平台自营 or 用户)

ALTER TABLE strategies ADD COLUMN seller_type VARCHAR(20) DEFAULT 'platform';
-- 'platform' | 'user' | 'kol'

ALTER TABLE strategies ADD COLUMN pricing_type VARCHAR(20) DEFAULT 'credits';
-- 'free' | 'credits' | 'subscription' | 'exchangeable'

ALTER TABLE strategies ADD COLUMN credits_price INT DEFAULT 0;
-- 积分价格

ALTER TABLE strategies ADD COLUMN is_exchangeable BOOLEAN DEFAULT FALSE;
-- 是否支持互换

ALTER TABLE strategies ADD COLUMN exchange_value INT DEFAULT 0;
-- 互换价值(积分)

ALTER TABLE strategies ADD COLUMN sales_count INT DEFAULT 0;
-- 销量

ALTER TABLE strategies ADD COLUMN rating_avg DECIMAL(3,2) DEFAULT 0;
-- 平均评分

ALTER TABLE strategies ADD COLUMN rating_count INT DEFAULT 0;
-- 评价数量

ALTER TABLE strategies ADD COLUMN refund_count INT DEFAULT 0;
-- 退款次数

ALTER TABLE strategies ADD COLUMN status VARCHAR(20) DEFAULT 'pending';
-- 'pending' | 'approved' | 'rejected' | 'archived'
```

#### 新建 `exchanges` 表(互换记录)
```sql
CREATE TABLE exchanges (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_a_id UUID REFERENCES auth.users(id) NOT NULL,
  user_b_id UUID REFERENCES auth.users(id) NOT NULL,
  strategy_a_id UUID REFERENCES strategies(id) NOT NULL,
  strategy_b_id UUID REFERENCES strategies(id) NOT NULL,
  status VARCHAR(20) DEFAULT 'pending',
  -- 'pending' | 'completed' | 'rejected'
  created_at TIMESTAMP DEFAULT NOW(),
  completed_at TIMESTAMP,
  INDEX(user_a_id),
  INDEX(user_b_id)
);
```

#### 新建 `bounties` 表(悬赏任务)
```sql
CREATE TABLE bounties (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  publisher_id UUID REFERENCES auth.users(id) NOT NULL,
  title VARCHAR(200) NOT NULL,
  description TEXT,
  reward_credits INT NOT NULL,
  requirements TEXT,
  deadline TIMESTAMP,
  status VARCHAR(20) DEFAULT 'open',
  -- 'open' | 'in_progress' | 'completed' | 'cancelled'
  winner_id UUID REFERENCES auth.users(id),
  submission_id UUID REFERENCES strategies(id),
  created_at TIMESTAMP DEFAULT NOW(),
  completed_at TIMESTAMP,
  INDEX(status),
  INDEX(publisher_id)
);
```

#### 新建 `subscriptions` 表(订阅服务)
```sql
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  subscriber_id UUID REFERENCES auth.users(id) NOT NULL,
  provider_id UUID REFERENCES auth.users(id) NOT NULL,
  plan_name VARCHAR(100),
  credits_per_month INT NOT NULL,
  status VARCHAR(20) DEFAULT 'active',
  -- 'active' | 'paused' | 'cancelled'
  current_period_start DATE DEFAULT CURRENT_DATE,
  current_period_end DATE,
  created_at TIMESTAMP DEFAULT NOW(),
  cancelled_at TIMESTAMP,
  INDEX(subscriber_id),
  INDEX(provider_id),
  UNIQUE(subscriber_id, provider_id, plan_name)
);
```

#### 新建 `user_reputation` 表(用户声望)
```sql
CREATE TABLE user_reputation (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) NOT NULL UNIQUE,
  reputation_score INT DEFAULT 0,
  -- 声望值
  reputation_level VARCHAR(20) DEFAULT 'newbie',
  -- 'newbie' | 'skilled' | 'expert' | 'master'
  total_strategies INT DEFAULT 0,
  -- 发布玩法数
  avg_rating DECIMAL(3,2) DEFAULT 0,
  -- 平均评分
  total_sales INT DEFAULT 0,
  -- 总销量
  total_refunds INT DEFAULT 0,
  -- 退款次数
  total_reports INT DEFAULT 0,
  -- 被举报次数
  updated_at TIMESTAMP DEFAULT NOW()
);
```

#### 新建 `strategy_reviews` 表(玩法评价)
```sql
CREATE TABLE strategy_reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  strategy_id UUID REFERENCES strategies(id) NOT NULL,
  user_id UUID REFERENCES auth.users(id) NOT NULL,
  rating INT CHECK (rating >= 1 AND rating <= 5),
  comment TEXT,
  is_verified_purchase BOOLEAN DEFAULT FALSE,
  -- 是否为真实购买用户
  helpful_count INT DEFAULT 0,
  -- 有帮助数
  created_at TIMESTAMP DEFAULT NOW(),
  INDEX(strategy_id),
  INDEX(user_id),
  UNIQUE(strategy_id, user_id)
);
```

### 5.2 API 设计

#### 1. POST /api/exchange/create
创建互换请求

**Request:**
```json
{
  "my_strategy_id": "uuid-a",
  "target_strategy_id": "uuid-b",
  "message": "想换你的玩法,我的也很不错"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "exchange_id": "uuid",
    "status": "pending",
    "expires_at": "2025-01-15T00:00:00Z"
  }
}
```

#### 2. POST /api/exchange/respond
响应互换请求

**Request:**
```json
{
  "exchange_id": "uuid",
  "action": "accept" // or "reject"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "exchange_id": "uuid",
    "status": "completed",
    "unlocked_strategy_id": "uuid-b",
    "credits_fee": 5
  }
}
```

#### 3. GET /api/exchange/matches
自动匹配可互换的玩法

**Query:**
```
?my_strategy_id=uuid-a
```

**Response:**
```json
{
  "success": true,
  "data": {
    "matches": [
      {
        "strategy_id": "uuid-b",
        "title": "Arbitrum 生态挖矿指南",
        "seller_name": "crypto_hunter",
        "value_difference": 5,
        "match_score": 0.95
      }
    ]
  }
}
```

#### 4. POST /api/bounty/create
发布悬赏任务

**Request:**
```json
{
  "title": "求 zkSync Era 最新空投攻略",
  "description": "需要包含所有测试网任务步骤",
  "reward_credits": 200,
  "deadline": "2025-01-20T00:00:00Z"
}
```

#### 5. POST /api/subscription/subscribe
订阅 KOL 服务

**Request:**
```json
{
  "provider_id": "uuid",
  "plan_name": "Pro",
  "credits_per_month": 150
}
```

## 六、核心差异化亮点

### vs 传统商城(Marketplace)

| 维度 | 传统商城 | 玩法交易所 |
|------|---------|----------|
| **交易主体** | 平台 ↔ 用户 | 用户 ↔ 用户,平台仅撮合 |
| **定价权** | 平台定价 | 用户自主定价 |
| **交易方式** | 仅现金/积分购买 | 购买 + 互换 + 悬赏 + 订阅 |
| **内容来源** | 主要是平台生产 | 70% UGC + 30% 平台 |
| **用户角色** | 消费者 | 消费者 + 生产者 |
| **社区属性** | 弱 | 强(互换需互动) |

### 核心创新点

1. **玩法互换机制** 🌟
   - 降低门槛:没积分也能换到玩法
   - 激励分享:用户主动贡献内容
   - 社交属性:促进用户互动

2. **悬赏驱动生产**
   - 需求导向:用户说了算
   - 快速响应:比平台自营更灵活
   - 质量保障:用户验收机制

3. **声望系统**
   - 防止刷单:低声望用户受限
   - 激励优质:高声望获得更多权益
   - 自治社区:用户参与治理

## 七、商业模式与盈利测算

### 7.1 收入来源

| 收入类型 | 占比(预估) | 说明 |
|---------|----------|------|
| **交易手续费** | 40% | 玩法购买/互换抽成 10% |
| **订阅分成** | 30% | KOL 订阅服务抽成 10-20% |
| **悬赏服务费** | 15% | 悬赏任务抽成 10% |
| **积分充值** | 10% | 用户购买积分($19 = 500 积分) |
| **广告位** | 5% | 首页推广位租赁 |

### 7.2 成本结构

| 成本项 | 月支出(美元) | 说明 |
|--------|-------------|------|
| AI 成本 | $500-$1,000 | 平台自营玩法生产 |
| 服务器 | $300-$500 | 数据库、CDN |
| 人力 | $3,000-$6,000 | 2 人运营(审核+客服) |
| 营销 | $1,000-$3,000 | 社群推广、KOL 合作 |
| **合计** | **$4,800-$10,500** | - |

### 7.3 盈利测算

**假设**:
- 月活用户: 1,000
- 人均月消费: 100 积分(= $4 充值 or 完成任务获得)
- 平台抽成: 10%

**收入**:
- 月交易额: 1,000 × 100 积分 × $0.04/积分 = $4,000
- 平台收入: $4,000 × 10% = $400(交易抽成)
- 积分充值: 1,000 × 20% × $4 = $800(假设 20% 用户充值)
- **月总收入**: $1,200

**盈亏平衡点**:
- 需月成本 $10,000 / 平台收入 = 至少 8,000 月活用户
