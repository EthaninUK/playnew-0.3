# 套利手册 - 产品设计方案 v2.0

**定位**：币圈套利知识库 + 精选实时机会展示
**英文名**：Arbitrage Handbook / Profit Playbook
**Slogan**：发现币圈的每一个价差机会

---

## 一、产品定位重新梳理

### 1.1 核心价值

**不是**：实时信号流（太技术化，用户看不懂）
**而是**：套利教育 + 机会展示（让用户理解价值，学会方法）

**用户画像**：
- 🎓 想学习套利但不知道从哪开始的新手
- 💡 知道套利但不知道有这么多类型的进阶用户
- 📚 想系统学习不同套利策略的认真学习者
- 🔍 偶尔想看看市场上有什么机会的兴趣用户

### 1.2 产品目标

1. **教育为主**：让用户理解 50+ 种套利方式的原理和操作
2. **机会展示**：精选 3-5 种主流套利的实时数据
3. **降低门槛**：用可视化和案例让复杂的套利变得易懂
4. **建立信任**：展示平台对套利的深度理解，吸引用户付费

## 五、技术实现建议

### 5.1 数据库设计

```sql
-- 套利类型表
CREATE TABLE arbitrage_types (
  id UUID PRIMARY KEY,
  category VARCHAR(50),           -- 10大分类
  name VARCHAR(100),              -- 套利名称
  slug VARCHAR(100) UNIQUE,
  difficulty INTEGER,             -- 1-5 难度
  avg_profit_min DECIMAL(5,2),    -- 收益范围
  avg_profit_max DECIMAL(5,2),
  risk_level INTEGER,             -- 1-5 风险
  frequency VARCHAR(50),          -- 出现频率描述
  description TEXT,               -- 简短描述
  principle TEXT,                 -- 原理说明（Markdown）
  guide TEXT,                     -- 操作流程（Markdown）
  case_study TEXT,                -- 真实案例（Markdown）
  risks TEXT,                     -- 风险分析（Markdown）
  tools JSONB,                    -- 推荐工具
  status VARCHAR(20),
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- 实时机会表（仅 3-5 种类型）
CREATE TABLE live_opportunities (
  id UUID PRIMARY KEY,
  type VARCHAR(50),               -- spot_arbitrage, funding_rate, depeg, etc.
  symbol VARCHAR(20),
  data JSONB,                     -- 具体数据（不同类型结构不同）
  profit_percent DECIMAL(8,4),
  detected_at TIMESTAMP,
  expires_at TIMESTAMP
);
```

## 九、设计要点总结

### 9.1 视觉设计
- **配色**：金色/绿色主调（代表盈利）
- **图标**：每种套利类型有独特图标
- **排版**：清晰的层级，大量留白
- **可视化**：流程图、收益图表、案例展示

### 9.2 用户体验
- **渐进式学习**：从简单到复杂
- **实用工具**：计算器、对比表
- **真实案例**：建立信任
- **清晰导航**：3 次点击内找到任何内容

### 9.3 内容策略
- **教育优先**：让用户学会，而不是依赖
- **机会展示**：证明平台价值
- **风险透明**：建立信任和专业形象
- **持续更新**：保持内容新鲜度

---

1. ✅ **降低认知门槛**：不是冷冰冰的数据流，而是有温度的教学内容
2. ✅ **建立专业形象**：50+ 类型展示你的深度
3. ✅ **实用价值**：计算器、工具、实时机会
4. ✅ **会员转化**：免费看原理，付费看机会
5. ✅ **内容资产**：文章可用于 SEO 和社交传播
