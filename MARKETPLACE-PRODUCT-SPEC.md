# 玩法商城产品方案 v1.0

## 一、产品定位与核心逻辑

### 1.1 产品定位
**Web3 垂直领域的"知识+工具"综合交易平台**
- 目标用户:加密货币投资者、DeFi 玩家、空投猎人、节点运营者
- 核心价值:将散落的玩法、工具、信号整合为标准化商品,降低信息壁垒
- 差异化:AI 自营起盘 → 真人合伙人共创 → 开放生态

### 1.2 商业模式画布

| 维度 | 内容 |
|------|------|
| **价值主张** | 标准化玩法商品、质量把关、托管交易、AI 持续产出 |
| **客户细分** | 小白(Guide/Task)、进阶(Script/Data)、专业(Service/订阅) |
| **渠道通路** | 站内推荐、主题导航、搜索、会员分层推送 |
| **客户关系** | 自助购买 + 社群支持 + VIP 服务 |
| **收入来源** | 佣金(5-20%)、会员费、增值服务、广告位 |
| **核心资源** | 12-24 个 AI Persona、内容生产流水线、风控体系 |
| **关键活动** | AI 内容生产、质量审核、纠纷仲裁、数据分析 |
| **重要伙伴** | 真人创作者、KOL、项目方、支付/托管服务 |
| **成本结构** | AI 成本、运营人力、服务器、营销获客 |

## 二、产品架构设计

### 2.1 六大核心品类定义

#### 品类 1:Guide(指南/课程/模板)
**定义**:结构化的知识交付物,帮助用户理解原理与操作步骤

| 属性 | 规格 |
|------|------|
| **交付形态** | 图文(Markdown/Notion)、PDF、视频(MP4)、模板(Excel/Airtable) |
| **定价区间** | $9-$199(单次购买) |
| **典型 SKU** | • 资金费率套利完全指南($49)<br>• 空投任务 SOP 合集($29)<br>• 项目尽调模板包($19)<br>• Dune 看板制作教程($79) |
| **质量标准** | • 字数 ≥3000 或视频 ≥30min<br>• 必须含实操截图/案例<br>• 测试题正确率 ≥80% 解锁 |
| **AI 生产流程** | 侦察(爬取文档)→ 结构化(GPT)→ 排版(Notion API)→ 质检(人工抽检 20%) |

---

#### 品类 2:Script/Tool(脚本/工作流/工具包)
**定义**:可执行的自动化程序或配置文件

| 属性 | 规格 |
|------|------|
| **交付形态** | n8n JSON、Python/JS 代码、Docker Compose、Chrome 插件 |
| **定价区间** | $29-$499(买断)或 $9/月(订阅) |
| **典型 SKU** | • 链上活跃度雷达(n8n Flow,$99)<br>• 套利路径探测脚本(Python,$199)<br>• 节点监控面板(Docker,$149)<br>• 批量钱包操作工具(JS,$79) |
| **质量标准** | • 必须提供安装文档<br>• 通过沙盒环境测试<br>• 明确依赖项与运行环境<br>• 提供 7 天售后支持 |
| **风险控制** | • 代码扫描(恶意调用、API Key 泄露)<br>• 用户协议:自行承担使用风险<br>• 禁止:钓鱼、盗币、攻击协议 |

---

#### 品类 3:Data/Signal(数据/信号/API)
**定义**:实时或周期性的信息流,用于决策辅助

| 属性 | 规格 |
|------|------|
| **交付形态** | API Token、Webhook 推送、Telegram Bot、Dune 看板链接、Excel 周报 |
| **定价模式** | 按月订阅($19-$299/月)或按调用计费($0.01/次) |
| **典型 SKU** | • 鲸鱼地址监控($49/月)<br>• 资金费差预警(Webhook,$29/月)<br>• 新池/新协议速递(TG Bot,$19/月)<br>• 巨鲸持仓看板(Dune,$99/月) |
| **SLA 承诺** | • 延迟 <5min(实时类)<br>• 准确率 ≥95%(可验证指标)<br>• 故障响应 <1h |
| **反作弊** | • Token 绑定设备/IP<br>• 并发限制<br>• 水印/指纹追踪(防转卖) |

---

#### 品类 4:Subscription(订阅/社群)
**定义**:持续的内容+服务组合,强调陪伴与更新

| 属性 | 规格 |
|------|------|
| **交付形态** | Discord/TG 社群、周报邮件、直播答疑、策略日历、工单支持 |
| **定价模式** | 分层订阅(Lite/Pro/Insider),月付/年付 |
| **典型 SKU** | • 空投情报站 Lite($29/月):周报+公开社群<br>• 套利策略 Pro($99/月):日报+私域群+月度复盘<br>• 全赛道 Insider($299/月):所有内容+1v1 咨询 |
| **续费机制** | • 前 7 天无理由退款<br>• 年付 8 折<br>• 邀请奖励(1 个月免费) |
| **内容更新** | • Lite:周更<br>• Pro:日更<br>• Insider:实时+专属 |

---

#### 品类 5:Service(顾问/代运营/部署)
**定义**:人力密集型服务,按项目交付

| 属性 | 规格 |
|------|------|
| **交付形态** | Milestone + SLA,平台托管资金(Escrow) |
| **定价模式** | 按小时($50-$200/h)或按项目($500-$5000) |
| **典型 SKU** | • 策略落地辅导(3 次 1v1,$299)<br>• 节点运维托管(月费,$399)<br>• 做市参数优化(按效果,$1500)<br>• 风控审查报告(单次,$799) |
| **交付流程** | 需求确认 → 报价 → 托管付款(30% 定金)→ 分阶段交付 → 验收 → 释放尾款 |
| **纠纷仲裁** | • 平台介入调查<br>• 证据提交(截图/录屏)<br>• 7 天裁决期<br>• 部分退款/重做 |

---

#### 品类 6:Task/Bounty(任务投放/悬赏)
**定义**:项目方或 KOL 发布,用户完成获得奖励

| 属性 | 规格 |
|------|------|
| **交付形态** | 链上交互、社交任务(转发/点赞)、UGC(文章/视频)、测试反馈 |
| **计费模式** | CPL(按线索)、CPA(按完成)、固定池瓜分 |
| **典型 SKU** | • 新链测试网任务(100 个名额,单价 $5)<br>• Twitter 推广(1000 转发,$2/次)<br>• 策略实战报告(悬赏 $200) |
| **反作弊** | • 钱包签名验证<br>• 社交账号年龄/粉丝数门槛<br>• 人工抽检(10%)<br>• 黑名单机制 |
| **结算周期** | 任务完成后 3-7 天,扣除平台费(10%)后发放 |

## 三、AI Persona 自营体系

### 3.1 Persona 设计原则
**12-24 个虚拟卖家,覆盖 9 主题 × 6 品类 ≈ 54 种组合(优先高频需求)**

#### Persona 模板示例:

**Persona #1:Airdrop Hunter Pro**
- **头像**:赛博朋克风格机器人
- **简介**:"专注空投套利 3 年,已完成 200+ 项目,平均 ROI 15x"
- **擅长品类**:Guide、Script、Task
- **覆盖主题**:A(空投)、E(新链)
- **典型商品**:
  - 《LayerZero 空投完全指南》(Guide,$49)
  - 多钱包批量交互脚本(Script,$99)
  - zkSync 生态任务包(Task,单价 $3)
- **生产节奏**:每周 2 个 Guide + 1 个 Script 更新

**Persona #2:DeFi Yield Scientist**
- **头像**:实验室科学家形象
- **简介**:"量化背景,专注 DeFi 收益优化与风险建模"
- **擅长品类**:Data、Subscription、Service
- **覆盖主题**:B(链上收益)、D(衍生品)
- **典型商品**:
  - 收益聚合器对比看板(Data,$29/月)
  - DeFi 周报订阅(Subscription,$99/月)
  - LP 策略优化咨询(Service,$1500/项目)
- **生产节奏**:日更数据 + 周更深度分析

### 3.2 AI 生产流水线(n8n 编排)

```
[侦察节点]
  ↓
  → Twitter/Discord/文档爬虫(每日)
  → 关键词提取(GPT-4)
  → 机会评分(自定义规则)
  ↓
[分析节点]
  ↓
  → 数据验证(链上查询/API 调用)
  → 可行性判断(历史成功率/成本收益)
  → 风险标注(合约审计状态/TVL 波动)
  ↓
[写作/编码节点]
  ↓
  → Guide:大纲生成 → 内容扩写 → 排版(Notion)
  → Script:代码框架 → 功能实现 → 测试用例
  → Data:API 对接 → 数据清洗 → 可视化
  ↓
[合规/风控节点]
  ↓
  → 敏感词过滤(法律/欺诈)
  → 代码安全扫描(Bandit/ESLint)
  → 收益承诺检查(禁止"保证"类表述)
  ↓
[定价节点]
  ↓
  → 成本核算(AI Token 成本 + 人工成本)
  → 竞品对标(类似商品价格中位数)
  → 动态定价(需求热度 × 稀缺度系数)
  ↓
[上架节点]
  ↓
  → 商品页生成(标题/描述/截图/标签)
  → SEO 优化(关键词密度/内链)
  → 推送通知(匹配用户兴趣标签)
  ↓
[营销节点]
  ↓
  → 首页推荐位(新品/爆款)
  → 社群推广(TG/Discord 机器人)
  → 联盟分销(邀请码 10% 返佣)
  ↓
[复盘节点]
  ↓
  → 销售数据分析(转化率/退款率)
  → 用户反馈收集(评分/评论)
  → 迭代优化(低分商品下架/高分商品扩展)
```

### 3.3 质量门槛设计

**三层过滤机制**:

1. **AI 自检(自动化,100% 覆盖)**
   - 字数/时长达标
   - 敏感词/风险提示
   - 格式规范(Markdown/代码注释)

2. **测试题解锁(用户端,强制)**
   - Guide 类:5 道选择题,答对 4 道才能购买/查看
   - Script 类:需声明已理解"代码风险"并签署协议
   - 目的:过滤纯小白,降低退款率

3. **人工抽检(运营端,20% 抽样)**
   - 新上架商品 100% 人工审核
   - 已上架商品:每周抽检 20%,重点关注低分/投诉商品
   - 建立"金牌/黑名单"Persona 机制

### 4.2 卖家后台(Persona 管理面板)

**AI Persona 自动化**:
- 运营者只需配置"生产计划"(每周产出类型、主题优先级)
- n8n 自动执行生产流水线
- 人工介入点:审核、定价微调、推广策略

**真人卖家入驻流程**(后期开放):
1. 申请认证(身份/专业背景)
2. 提交样品(1-2 个商品)
3. 通过审核(质量评分 ≥4.0)
4. 签署协议(平台抽佣 15-20%,纠纷处理条款)
5. 开店上架

### 4.3 交易流程设计

#### 数字商品(Guide/Script/Data)
```
用户下单 → 支付(Stripe/Crypto) → 立即交付 → 7 天内可申请退款(需说明理由)
```

#### 订阅类(Subscription)
```
选择套餐 → 首次支付 → 加入社群/获得 Token → 自动续费(可随时取消)
```

#### 服务类(Service)
```
提交需求 → 卖家报价 → 托管付款(30% 定金)→ Milestone 交付 → 验收 → 释放尾款
```

#### 任务类(Task/Bounty)
```
任务发布(项目方托管资金)→ 用户完成(提交凭证)→ 审核(自动/人工)→ 发放奖励
```

### 4.4 风控与仲裁机制

**风险分级标签**(强制展示):
- 🟢 低风险:纯知识类,无资金操作
- 🟡 中风险:需授权钱包/API,资金 <$1000
- 🔴 高风险:涉及大额资金、合约交互、杠杆操作

**纠纷处理流程**:
1. 用户发起投诉(需提供证据:截图/录屏/交易哈希)
2. 平台冻结争议部分资金
3. 双方举证期(3 天)
4. 平台裁决(7 天内):
   - 卖家责任:全额退款 + 警告/下架
   - 用户责任:驳回申诉,扣除手续费
   - 双方责任:部分退款 + 商品改进
5. 不满裁决可申请二次人工复审

**反作弊机制**:
- 刷单检测:同 IP/设备异常购买
- 虚假评价:短时间大量好评/差评
- 转卖防护:API Token 设备绑定、内容水印

## 五、商业模式与财务测算

### 5.1 收入结构

| 收入来源 | 占比(预估) | 客单价 | 备注 |
|---------|----------|--------|------|
| 商品佣金 | 50% | 5-20% 抽成 | AI 自营 100% 归平台,真人卖家后期 15-20% |
| 会员订阅 | 30% | $29-$299/月 | 分层会员(Lite/Pro/Insider) |
| 任务服务费 | 10% | 10% 抽成 | 项目方发布任务,平台收取服务费 |
| 广告位 | 5% | $500-$2000/月 | 首页/分类页推广位 |
| 增值服务 | 5% | $99-$999 | 数据 API、白标服务、企业定制 |

### 5.2 成本结构

| 成本项 | 月支出(美元) | 备注 |
|--------|------------|------|
| AI 成本(GPT-4/Claude) | $2,000-$5,000 | 12-24 Persona 持续生产 |
| 服务器/基础设施 | $500-$1,000 | 数据库、CDN、n8n 托管 |
| 人力成本 | $8,000-$15,000 | 2-3 人运营团队(审核、客服、BD) |
| 支付手续费 | 3-5% 流水 | Stripe/Crypto 支付 |
| 营销获客 | $3,000-$10,000 | KOL 合作、社群推广、SEO |
| **合计** | **$13,500-$31,000** | 不含真人卖家分润 |

### 5.3 盈亏平衡测算

**假设**:
- 平均客单价:$50
- 平台抽成:20%(前期 AI 自营 100%)
- 月活用户:5,000
- 购买转化率:5%
- 复购率(订阅):30%

**收入测算**:
- 月交易额:5,000 × 5% × $50 = $12,500
- 平台收入(AI 自营):$12,500
- 订阅收入:5,000 × 30% × $99 = $148,500(假设 30% 用户订阅 Pro)
- **月总收入**:$161,000

**盈亏平衡点**:
- 月成本 $31,000 / 平均客单价 $50 / 抽成 20% = 需月交易额 $155,000
- 对应月活 5,000 + 5% 转化率 + 客单价 $50 的情况下可盈利

## 九、技术架构建议

### 9.1 技术栈选型

**前端**:
- Next.js 14(App Router) + TypeScript
- Tailwind CSS + shadcn/ui
- React Query(数据缓存)

**后端**:
- Directus(无头 CMS,管理商品/用户/订单)
- PostgreSQL(主数据库)
- Redis(缓存/队列)
- n8n(AI 生产流水线编排)

**AI 层**:
- OpenAI GPT-4(内容生产)
- Anthropic Claude(代码生产/审核)
- Langchain(工作流编排)

**支付**:
- Stripe(信用卡/订阅)
- Crypto 支付(USDT/USDC via Coinbase Commerce)

**存储**:
- Supabase Storage(PDF/视频/代码包)
- Notion API(Guide 类内容托管)

**搜索**:
- Meilisearch(已部署,商品搜索)

### 9.2 数据库核心表设计

**商品表(products)**
```sql
id, title, slug, category_id, type(guide/script/data/subscription/service/task),
persona_id, price, pricing_model(one_time/monthly/hourly),
description, content_url, risk_level, ai_generated(bool),
sales_count, rating, status(draft/published/archived),
created_at, updated_at
```

**订单表(orders)**
```sql
id, user_id, product_id, amount, payment_method, payment_status,
delivery_status, refund_status, escrow_id(Service 类托管),
created_at, completed_at
```

**订阅表(subscriptions)**
```sql
id, user_id, product_id, plan(lite/pro/insider), status(active/cancelled),
current_period_start, current_period_end, stripe_subscription_id
```

**Persona 表(personas)**
```sql
id, name, avatar, bio, specialty_categories[], specialty_types[],
production_schedule(JSON), quality_score, is_active
```

**任务表(tasks)**
```sql
id, publisher_id, title, type(cpl/cpa), reward_pool, reward_per_unit,
requirements(JSON), verification_method, status, deadline
```
